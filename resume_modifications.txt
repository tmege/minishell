Fix memory leaks : '.' et '..' en entree (21/02/2026)
======================================================

Cause : '.' et '..' ne contiennent pas '/', donc find_cmd_path() les cherche
dans le PATH et trouve un dossier accessible (ex: /usr/bin/.). execve() echoue
avec EACCES (Permission denied), mais le process enfant faisait exit() sans
rien liberer.

Fichiers modifies :

1. src/executor.c - exec_external() (child fork pour commande externe)
   - Sur echec apply_redirections() : free_cmds + free_envp + free(path) avant exit(1)
   - Sur echec execve()             : free_cmds + free_envp + free(path) avant exit(126)

2. src/executor_cmd.c - pipe_exec_cmd() (child pipe commande externe)
   - Sur echec execve()     : free_cmds + free_envp avant exit(126)
   - Sur commande introuvable : free_cmds + free_envp avant exit(exit_status)

3. src/executor_pipe.c - pipe_child() (child pipe sorties anticipees)
   - Sur echec apply_redirections() : free_cmds + free_envp avant exit(1)
   - Sur cmd->args NULL             : free_cmds + free_envp avant exit(0)

Resultat valgrind : 0 definitely lost, 0 indirectly lost
