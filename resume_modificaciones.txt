Fix memory leaks : '.' y '..' como entrada (21/02/2026)
=========================================================

Causa : '.' y '..' no contienen '/', por lo que find_cmd_path() los busca
en el PATH y encuentra un directorio accesible (ej: /usr/bin/.). execve() falla
con EACCES (Permission denied), pero el proceso hijo hacia exit() sin
liberar nada.

Archivos modificados :

1. src/executor.c - exec_external() (fork hijo para comando externo)
   - En fallo de apply_redirections() : free_cmds + free_envp + free(path) antes de exit(1)
   - En fallo de execve()             : free_cmds + free_envp + free(path) antes de exit(126)

2. src/executor_cmd.c - pipe_exec_cmd() (hijo de pipe comando externo)
   - En fallo de execve()      : free_cmds + free_envp antes de exit(126)
   - En comando no encontrado  : free_cmds + free_envp antes de exit(exit_status)

3. src/executor_pipe.c - pipe_child() (hijo de pipe salidas anticipadas)
   - En fallo de apply_redirections() : free_cmds + free_envp antes de exit(1)
   - Con cmd->args NULL               : free_cmds + free_envp antes de exit(0)

Resultado valgrind : 0 definitely lost, 0 indirectly lost
